### INKIND

```json
{
   {
        "providerDeliveryId":"e05b049a-a93a-4b04-af96-5824e0af6684",
        "deliveryAmount":"1.00",
        "deliveryUnit":"pcs",
        "deliveryVoucherSerialNo": "500000650388213UR",
        "deliveryReferenceNumber": "",
        "status":"DELIVERY_COLLECTED"
   },
   {
        "providerDeliveryId":"7cbba01e-df8f-406e-8bcf-8154994a16d4",
        "deliveryAmount":"2.00",
        "deliveryUnit":"pcs",
        "deliveryVoucherSerialNo": "500000435643513UR",
        "deliveryReferenceNumber": "",
        "status":"DELIVERY_COLLECTED"
   }
}
```

- DELIVERY_COLLECTED -> Beneficiary collected delivery or cash is delivered via iban
- DELIVERY_PENDING -> Wait step which delivery provider ensures that delivery is collected
- DELIVERY_REVERTED -> Delivery or cash is reverted
- DELIVERED -> Delivery request is committed to the delivery provider

## Response Payload

### 200 Response

A successful callback response from the Service Provider’s callback service.

```json
{
    "status": "success",
    "message": "Callback successfully received",
    "data": {}
}
```

### 400 Response

If the validation fails or there is a bad request, the response will be:

```json
{
    "status": "failed",
    "error": {
        "code": 400,
        "message": "Validation failed. Please check your input.",
        "type": "BadRequest"
    }
}
```

### 401 Response

If the verification fails, the response will be:

```json
{
    "status": "failed",
    "error": {
        "code": 401,
        "message": "Verification failed",
        "type": "Unauthorized"
    }
}
```

### 500 Response

In case of an internal server error, the response will be:

```json
{
    "status": "failed",
    "error": {
        "code": 500,
        "message": "Internal Server Error",
        "type": "Exception"
    }
}
```
